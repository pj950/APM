# AI 相镜 (AI Mirror)

数字人格解析装置 — 基于计算机视觉与生成式 AI 的交互艺术装置。

## 技术栈

| 领域 | 技术 |
|------|------|
| 框架 | React 18 + Vite 5 |
| 状态管理 | Zustand |
| 视觉追踪 | @mediapipe/tasks-vision (当前为 Worker 双路检测，实时回传 face + pose) |
| 3D 渲染 | Three.js + React Three Fiber |
| 着色器 | GLSL (自定义粒子系统) |
| 大模型 | SSE 流式调用 (通义千问/OpenAI 兼容) |

## 状态机流转

```
STANDBY → SCANNING → QUESTIONING → GENERATING → RESULT → STANDBY
```

- RESULT 阶段 30s 后自动回退 STANDBY
- QUESTIONING 阶段会停留在当前题，直到用户明确作答或主动返回

## 开发

```bash
cd ai-mirror
npm run dev        # 启动开发服务器 (localhost:5173)
npm run build      # 生产构建
```

开发服务器固定使用 `localhost:5173`，并启用 Vite `strictPort`；如果 5173 已被占用，服务会直接报错，不会自动跳到 5174/5175/5176。

`public/model/` 与 `public/images/` 体积较大，作为本地资源目录保留，不随 Git 推送；需要这些模型或人格图片时，从本机备份复制到对应目录即可。

## 演示页面（待机流体模式入口）

在 `STANDBY` 流体模式下提供三个独立演示入口，均不参与主状态机叙事：

- **💧 水流模拟**（`WATER_DEMO`）：流体/水面模拟演示。
- **🤏 橡胶手指拉伸**（`GUMGUM_DEMO`）：基于 MediaPipe 手部关键点的“拉伸手指”互动。
- **🎭 面部追踪演示**（`FACE_DEMO`）：基于 **MindAR (WebAR)** 的 3D 人脸面具追踪。

### 面部追踪演示（MindAR）

参考 `dilmerv/FaceTrackingDemo` 的效果，用 Web 技术复刻“面具实时贴脸并跟随头部姿态”的体验：

- 使用 [`mind-ar`](https://github.com/hiukim/mind-ar-js)（MIT，基于 TensorFlow.js + MediaPipe face mesh）独占摄像头并自带 Three.js 渲染。
- 进入 `FACE_DEMO` 后，CV 主管线（`useCVCapture`）会主动让出摄像头（`FACE_DEMO` 已从 `isCameraStageActive` 移除），避免与 MindAR 抢占设备。
- `FACE_DEMO` 已集成 6 题二选一答题流程：页面保留 MindAR face mesh 面具，同时在顶部显示左右选项、底部显示当前题；不使用举手识别，直接读取 MindAR 面部网格的头部 yaw，用户看向左侧/右侧并保持约 1.5 秒即可确认对应选项，答完后写入 6 维人格并进入生成阶段。
- 左右语义已校正为“看左选左、看右选右”（按用户视角），避免镜像预览和世界坐标符号差异导致反选。
- `FACE_DEMO` 的确认反馈已升级为“看向即开火”：看向某侧并进入保持阶段时，会先出现红色预瞄激光；达到阈值并完成保持后，立即触发更亮更粗的红色命中激光，同时被命中的答案卡触发崩碎动画，再进入下一题，避免无感跳题。
- 为降低误触，确认阈值已提高（yaw 约 ±0.28），并保留更清晰的“保持确认”提示。
- 为保证不同屏幕和帧率下都清晰可见，激光采用“父层定位 + 子层 beam/flash 动画”结构，且延长了可见时长并提高亮度，避免动画覆盖位移或帧率波动导致光束难以察觉。
- **直接复用 `dilmerv/FaceTrackingDemo` 的 Unity 原始面具贴图**：该 Unity 项目的“面具”并不是 3D 模型，而是贴在 AR 人脸网格上的 2D 面部贴图（`Assets/Textures/` 下的 `cartoon` / `humanface` / `virus1` / `virus2` / `superheros` / `uv`）。这些贴图已下载到 `public/unity-face/textures/`；当前自动轮换只启用 `cartoon` / `superheros` / `uv`，暂不轮换 `humanface` / `virus1` / `virus2`。
- 实现上用 MindAR 的人脸网格（`addFaceMesh()`）作为载体，把这些贴图 UV 映射到实时追踪到的脸上；所有贴图预加载，切换时只替换材质贴图 uniform，每 5 秒自动轮换一张（对齐 Unity 里 `ToggleFace` 的切换演示），零延迟。
- `cartoon` 路径使用了自定义 ShaderMaterial 复刻 `CartoonAnimated`：核心为 UV 绕中心随时间连续旋转，并做 ×2 emission 提亮。
- 原 Unity 材质是 Opaque / Alpha=1 / AlphaClip=0，贴图本身没有透明眼嘴；Web 版同样不做眼睛和嘴巴开口，保留完整动态 face mesh 覆盖，避免不规则裁切造成“洞口不贴合”的观感。
- 为贴近原 Unity 设置，FACE_DEMO 使用 MindAR 默认 OneEuro 滤波参数，避免过度平滑造成面具滞后；材质主体改为 Three.js `MeshStandardMaterial`，对应 Unity Universal PBR 的 Opaque + ZWrite + `_Metallic=1` / `_Smoothness=0.695` 观感。
- 原项目的 `AR Default Face Animated 1.prefab` 由 `ARFaceManager` 生成动态 face mesh，并把 `CartoonAnimated_1.mat` 直接贴到这张每帧更新的网格上；Web 版对应使用 MindAR 的 `addFaceMesh()`，其 geometry 会随 `metricLandmarks` 每帧更新。
- `CartoonAnimated_1.shadergraph` 的核心是在 PBR 材质里执行 `Time -> Rotate(UV, 0.5) -> SampleTexture2D -> × Color -> ×2`；Web 版现在去掉额外红色 tint，以白色材质基底直接显示原贴图颜色，并通过替换 Three `map_fragment` 保留 UV Rotate/texture ×2，保留 Three 标准 PBR 光照、环境反射、金属度和粗糙度来提供立体感。
- MindAR 默认 face mesh 的上边界比 ARFoundation/ARKit 脸网格更保守，容易露出额头上沿；Web 版在高额头 UV 区域对动态顶点做轻微上扩，补足额头覆盖范围。
- **不要覆盖 MindAR 自动注入的 video/canvas 尺寸与偏移样式**（例如 `object-fit: cover` 或 `width/height: 100%`）；MindAR 会在 `_resize` 中按相机参数计算对齐，覆盖后会出现“面具像蒙上去、不贴脸”的错位感。

> 实现要点：`mind-ar` 无 TS 类型声明，由 `src/mind-ar.d.ts` 为深导入路径补充 `MindARThree` 类型；并在 `vite.config.ts` 的 `optimizeDeps.exclude` 中排除 `mind-ar`（其内部 worker / 动态加载会导致 Vite 依赖预构建失败）。

## 流程机制

### 自动进入与离开
- **自动进入扫描**：STANDBY 阶段会持续监听 CV 信号。当用户站在屏幕前方（检测到有效骨架点）且注意力分数 > 0.4 时，会在 600ms 延迟后自动进入 SCANNING 阶段，无需手动点击。
- **手动进入扫描**：点击"开始扫描"按钮可随时立即进入。
- **SCANNING 中途退出**：在扫描页面点击"← 返回"按钮可随时退回 STANDBY 阶段。
- **自动离开检测**：在 SCANNING 阶段若用户离开屏幕（连续 3 秒无有效骨架点且注意力 < 0.2），系统会自动回到 STANDBY 阶段。

### 待机开场
- **太阳系待机场景**：`STANDBY` 不再只是抽象线框球体，而是切换为更接近天文馆质感的太阳系开场，包含恒星背景、深空雾团、发光太阳、八大行星轨道和小行星带。
- **无文字待机入口**：待机前景移除了 `AI 相镜` 标题、说明文字和传统按钮，用户直接在太阳系里点击地球即可开始，不再被额外 UI 文案打断。
- **压缩但更真实的公转**：行星轨道改为带偏心率与倾角的椭圆轨道，并按真实公转周期做压缩映射，保证肉眼可见的动态同时保留内外行星快慢差异。
- **更写实的天体材质**：太阳表面加入分层热斑与日冕光晕；类岩质、类气态、云层型和冰巨星会分别生成不同的程序贴图，并叠加粗糙度、凹凸、大气壳或云层外壳，让各行星不再只是“同一颗球换颜色”。
- **重点行星微调**：地球额外强化了海陆反差、极冠与云层层次；木星补强了多层云带和大红斑周围的扰动；土星环改成带分层透明度与环缝纹理的材质，在远景里也更容易一眼分辨。
- **更绚烂的深空背景**：待机背景新增了发光星云层、亮星精灵层和更分明的远近星场，背景不再只是稀疏点状星星，而会有更明显的星光、色彩层次和空间纵深感。
- **银河带增强**：背景里额外加入了一条斜穿画面的银河带和沿带分布的恒星尘埃层，让天空不只是均匀散点，而会出现更接近真实深空摄影的亮带结构与密集星区。
- **地球穿越过场**：点击地球后会触发独立的镜头推进序列，先快速拉近地球，再穿入大气层，最后落到近地表与海洋视角，然后自然切入问答阶段。
- **落地段细节增强**：地球穿越的最后阶段补入了更亮的近地表光照、海面高光层、近地云层层和更厚的雾化地平线，让海洋与云层不再只是一层平面贴图，而会有更明显的层次和亮度。
- **海面与云影增强**：近地表海面现在额外加入了程序化波纹法线和移动的反射扫光，近地云层下方也补入了会跟随偏移的云影层，让“穿过云层后落向海面”的段落更像真实俯冲而不是静态贴片。
- **镜面扫光与云影深化**：海面反射进一步收紧为更窄、更强的太阳镜面亮带，云影则改成带软边和深浅起伏的程序化阴影纹理，因此掠海瞬间的高光更集中，云团压在海面上的体积感也更强。
- **天空与海岸过渡增强**：地球穿越末段不再一直停留在“太空球体 + 发光平面”的抽象结构里，而会把背景逐步过渡成更接近真实大气层颜色的天空壳，同时在海面尽头补入近地海岸/陆地层，让最后看到的更像俯冲进入地球环境，而不是悬浮在 CG 球体前。
- **卫星式近地表贴图**：地球穿越落地段的陆地层改成了更接近卫星俯视的海岸线与海陆色带生成方式，不再只是简单色块或条带，因此低空看到的海岸边缘、浅海过渡和山地明暗会更像真实航拍地表。
- **分层云切片穿云感**：单一云层平面之外，现在额外叠了多层不同高度、尺度和漂移速度的云片，镜头下冲时会先后穿过前后云层，最后进入近地表云带，整体更接近真正“钻进大气层”的感觉。
- **WebGL 探测与创建放宽**：除了非破坏式复检之外，WebGL 现在还会按“`webgl2` 标准配置 -> `webgl` 标准配置 -> `webgl` 最小兼容配置 -> `experimental-webgl`”这条链路逐级回退；检测和真实 `Canvas` 创建都走同一套 best-effort 逻辑，避免出现“探测说能用，真正渲染又起不来”或“标准配置起不来，但兼容配置其实能跑”的割裂。
- **兼容模式景深增强**：兼容模式太阳系不再只是倾斜椭圆轨道图，而是给各轨道加了前后景深、远近缩放、亮度起伏和前后景雾层，让行星在 CSS 公转时也会有更明确的前后穿插感，视觉上更接近 3D 开场而不是平面排版。
- **兼容模式点击地球过场**：对于拿不到 WebGL 上下文的设备，点击地球后的兼容模式不再只是“地球放大 + 其他元素淡出”，而是改成了分段式 CSS 电影过场：地球本体缩放、环形冲击、大气壳、穿云遮挡、中心曝光和末段地平线会依次接管画面，让 fallback 环境下的进入流程也更像真正俯冲入场。
- **观测者坍缩过场**：当前待机页点击地球后，WebGL 路径不再执行“进入地球”的俯冲，而是切换成 Observer Collapse 过场：太阳、行星、背景星场和碎片会沿各自轨道向中心奇点同步塌缩，相机也会先拉开观察再向奇点压缩；因此演示重点从“飞进一颗星球”转成“整个太阳系向观察者中心坍塌”。
- **问卷显形改成奇点撕开**：坍缩结束后不再把问卷页面整块硬切进来，而是先用一层短暂的奇点撕裂显形把中心打开，再让题面、手势引导和读题界面从中心长出来；因此进入 QUESTIONING 的第一感受不再像切页面，而更像奇点把面具和问题一起吐出来。
- **摄像头启动竞态补偿**：相机启动不再依赖“刚好在 worker ready 后命中一次 `startCamera`”；进入 SCANNING / QUESTIONING 后现在会主动检查是否已经拿到 live track，若没有则在短时间内自动补启动几次，因此即使发生“先切阶段、后 ready”或一次性启动时机错过，也不应该再出现摄像头一直不点亮的死锁。
- **AR 级贴脸面具接入 QUESTIONING**：QUESTIONING 阶段现在会直接消费 MediaPipe 的 face landmarks，把赛博面具挂到镜像摄像头视频上，而不是继续漂在背景正中央；面具会跟随脸部位置、尺寸和 roll 旋转移动，并用近似 yaw / pitch 驱动内部 3D 角度，因此观众在转头、侧脸和前后靠近时，会更像在戴一个实时数字滤镜而不是看一个悬浮摆件。
- **贴脸面具容错增强**：面具挂载不再要求脸必须离镜头很近；当前会接受更小的人脸尺寸，并在 face landmarks 短暂抖动或漏一两帧时保留上一帧有效变换一小段时间，避免出现“摄像头明明开着，但面具时有时无甚至完全不出现”的断崖式闪断。
- **QUESTIONING 改为姿态优先调度**：为避免贴脸面具的人脸追踪挤占抬臂作答链路，worker 现在会优先跑 pose，再低频补 face，并暂时关闭问答阶段未使用的 hand tracking；这样举手确认应当比上一版更容易被稳定捕捉。
- **手势选择升级成能量爆破**：QUESTIONING 阶段不再只有按钮进度环和骨骼引导线；当用户抬起左臂或右臂完成确认时，会在对应腕点或选项位置释放一圈赛博冲击波，并叠加短促的屏幕震颤与高亮爆闪，让“作答”从 UI 反馈变成像在释放指令或法术。
- **TTS 接入全息语音波纹环**：语音服务现在会导出实时音量包络，QUESTIONING / DIALOGUE 里的 3D 面具外围也随之挂上一圈全息环形波纹；AI 朗读时圆环会根据真实音频能量扩张、起伏和闪烁，而不是只靠 `isSpeaking` 做统一呼吸，因此声音第一次真正拥有了“物理形态”。
- **问卷加入赛博故障脉冲**：切题、AI 开口和确认答案这三个关键节拍点，现在都会触发一小段 RGB split、扫描线和故障噪声脉冲；这层效果持续时间很短，只负责把节奏点敲响，强化“系统正在运算/入侵/回显”的黑客质感，而不会长期污染画面。
- **椭圆装饰层清理**：待机太阳系里原本额外加了几层椭圆星云片，坍缩过场里也有一组容易被看成“贴上去图案”的椭圆辉光与径向装饰；这一轮把这些最抢画面的椭圆装饰层关掉，同时把行星轨道恢复为更淡的引导线，只保留必要的轨道信息，不再让装饰元素抢画面。
- **旧 EarthDive 路径正式清退**：既然当前主叙事已经切到 Observer Collapse，这一轮把旧的 EarthDive 运行序列、专属贴图缓存、预热逻辑和相关 helper 从代码里物理删除，只保留仍被轨道行星材质复用的通用云层贴图生成能力，避免仓库里继续挂着一整套不会再执行的历史路径。
- **姿态联动场景**：待机时用户的双臂距离、抬手高度和身体倾斜会继续影响轨道展开、时间流速、整体倾角和镜头轻微升降，让开场不是纯循环背景而是可被人“拨动”的场景。

### 问卷交互
- 逐题展示 6 个二选一人格维度问题，对应 capital / spirit / intellect / social / order / energy。
- 每题进入后会自动中文读题，并提示“抬左臂选左侧 / 抬右臂选右侧”；也支持手动点击“重读题目”。
- 问答页会显示 5 种声音预设和对应的动态面具，自动读题时会随机切换音色与面具风格。
- 贴脸面具加载会从 `public/model` 目录中随机选择 AR 友好模型（优先过滤头盔/整头类文件名，保留面具类）；当前不再启用 3D 粒子脸作为默认回退展示。
- QUESTIONING 贴脸层已放大为更激进的自适应尺寸区间，并在 AR 跟随模式下关闭深色兜帽背板，减少“中间黑框”观感。
- 支持姿态识别抬臂作答，要求先双臂回到中性位，再保持目标手臂约 1.2 秒完成确认。
- 抬臂选择加入了抗抖容错：轻微姿态抖动、短暂漏帧或手腕只比肩膀高一点时，不会立刻把选择进度打断。
- 抬臂作答与手势 inspect 已拆成两条输入模式：进入举臂确认时会冻结 portal 的手势驱动，只有在 pose 处于 ready 且没有答题 hold 时，手部开合才会接管中央 artifact 的 inspect 动效。
- 6 道题不再只是同一套面板重复切换，而是被组织成 6 个连续 chapter scene；每切到新题时，顶部会重播对应的场景标题层与氛围描述，下方章节轨道也会同步推进，让问卷更像一幕幕展开的交互仪式。
- 问答页中央视觉已升级为多层 `portal artifact`：会围绕当前维度生成孔径、环带、粒子轨道与标签带，让视觉焦点稳定锁在中央“入口圣物”而不是松散背景。
- portal aperture 已从单一发光核改成 `liquid core`：中央现在带有液态膜、分层波纹与高光液面，inspect 时会继续跟随手势改变粘度感、涨缩与折射观感。
- `liquid core` 已增强为持续可读的液态表面：液膜层被前置、核心高光压低，并加入常驻的表面流动与高光位移，不再只像一层被盖住的静态发光球。
- 中央 liquid shader 现已从“小内核”扩到整个 `portal aperture`：QUESTIONING 不需要整页切到 3D 场景，而是在主孔径内部直接用独立 R3F canvas 渲染液态流场；无 WebGL 时也不再回退到旧的圆形 core，而是退回到更接近液膜的兼容层。
- QUESTIONING 液态主视觉已回到“整屏粒子流体漂浮”：当前不再使用杯体/槽体容器，画面改为全屏漂移的流体纹理 + 粒子/波纹叠层，整体观感更接近在空间里飘动的流体场。
- 每道题都会切换不同流体样式：6 个维度（capital / spirit / intellect / social / order / energy）会驱动不同的流体频率、色相和粒子风格，避免每题都像同一张背景。
- 流体保持可交互：手势 inspect 的 openness / grip / energy 会继续联动 shader 流速、粒子密度和波纹强度，让用户动作能实时“拨动”流体状态。
- QUESTIONING 当前已启用性能优先的极简模式：暂时移除中央与背景特效层（液态、波纹、冲击波、故障、章节显形等），只保留题目、按钮与举手识别确认链路，优先保证现场流畅度。
- 在极简模式下，抬手作答的左右引导虚线与实时手臂检测线已恢复显示，用于明确提示“左臂选左 / 右臂选右”，同时仍不启用高开销视觉特效。
- 抬手作答判定已增加防误触门槛：仅在关键点可见性充足、手腕与肘部同时达到抬升阈值且持续稳定短时间后才进入确认进度；中立复位也要求双臂明确回落，减少“未举手就被选择”的误判。
- QUESTIONING 中央 portal 的旋转不再通过 `requestAnimationFrame + React setState` 每帧重渲整页，而是改成直接写 CSS 变量驱动；这一轮主要是为了解掉“中间变化不大同时还掉帧”的共同根因。
- portal 的 inspect 不再只影响整体转动，而会继续驱动 beam、ring、aperture、core、glyph band、orbiters 等分层响应，让中央核心更像可被“检视”的仪式装置而不是单颗球体。
- QUESTIONING 入场已改为 `awakening` 三段：先奇点收束，再唤醒中央 portal，最后让题面、选项与引导线从中心长出；读题会在入场完成后才开始。
- 每题切换时的 scene overlay 也加入了 `liquid unfold`：会先生成一层液态门膜与裂缝，再把当前章节标题和氛围文字从膜后顶出来，让 6 幕之间的切换不再只是文字换页。
- 6 幕的 `liquid unfold` 现在各自拥有不同的膜宽、圆角、裂缝高度与流动速度，切题时不再是同一张液态卡片重复播放，而会跟随维度切换不同的液体几何。
- 问答页保持单屏构图：中央 portal 固定在舞台中心，题面停留在底部信息板，左右答案分布在上方两侧，避免主视觉被内容面板挤偏。
- 支持"← 上一题"按钮回退到前一题。
- 支持"返回"按钮随时退出问卷回到 STANDBY 阶段。
- 答完 6 题自动进入 GENERATING 阶段。

### Archetype 派生与 LLM 生成
- 根据三个问卷答案统计用户倾向的 trait（Order / Chaos / Intuition / Logic / Flora）。
- 多个 trait 等分时，会优先选择 Flora（自然生长力），其次按组合关系推导：
  - Flora → Flora
  - Chaos + Intuition → Singularity（打破秩序的直觉）
  - Chaos 单独领先 → Plasma（躁动能量）
  - Intuition → Nebula（直觉星云）
  - Order / Logic → Crystal（理性晶体）
- `modifierType` 根据扫描时的 CV 实时值派生：
  - movement > 65% 且 attention > 55% → Volatile（活跃专注）
  - attention > 75% → Resonant（极度专注）
  - movement > 60% → Volatile（高活跃）
  - movement < 30% 且 attention < 40% → Drifting（低活跃）
  - 其他情况 → Static（静止）
- **LLM 人格差异化**：根据 baseType 动态调整 prompt 框架（Crystal 用逻辑推理、Nebula 用直觉感知、Plasma 用能量表述、Flora 用生态隐喻、Singularity 用虚无反转），使得生成的判词与人格底色相匹配，而不是统一的风格。

### 结果页展示与分享
- 显示推导出的 Archetype 徽章（baseType × modifierType）。
- 展示 LLM 生成的人格判词。
- 详情面板包含：
  - 扫描总耗时（从进入 SCANNING 到生成 RESULT 的秒数）。
  - CV 最终指标快照：活跃度、微笑值、专注度。
  - 问卷选择回顾（Q1/Q2/Q3 各自选择的选项标签）。
- **结果分享与导出**：
  - 点击"📋 分享"按钮，将完整结果（Archetype、判词、CV 数据、问卷回顾）复制到剪贴板。
  - 点击"💾 导出"按钮，将结果下载为本地 txt 文件。

## 当前扫描策略

- CV 检测当前运行在 WebWorker 中，`face` / `pose` 分别做节流检测并回传主线程，优先保证扫描阶段流畅度。
- `smile / attention` 优先基于真实脸部 landmarks 计算；当脸部暂时不可用时，会退化为基于 pose 的粗略注意力估计。
- 目前已将脸部精细识别通过开关暂时注销，但代码仍保留在仓库中；当前扫描优先使用头部/上半身轮廓来填充粒子。
- 摄像头生命周期现在由 `useCVCapture` 直接管理：进入 `SCANNING / QUESTIONING` 会自动启动并重挂视频流，离开这两个阶段会主动停止，避免只切换 UI 但摄像头未真正激活。
- 当摄像头启动失败时，扫描页会直接显示更具体的失败原因（权限拒绝、设备不存在、设备被占用等），并提供“重新启动摄像头”按钮用于手动恢复。
- 扫描层现在也接入了人格粒子 preset：会优先使用 `calculatedArchetype`，若尚未生成最终人格，则基于实时 CV 数据推一个 preview archetype。
- `baseType` 会影响扫描粒子的主辅色和整体观感，`modifierType` 会影响密度、发光、扩散、弹性和扫描速度，因此扫描粒子不再只有一种固定样式。
- 扫描时会把人体骨骼轮廓与头脸大致轮廓叠加显示；脸部不再展开 face mesh 细节，只保留整体外形。
- 头部保留顶部外壳弧线和少量宽松填充粒子，用来给出头脸的大致轮廓，而不是展示五官细节。
- 身体部分不再只沿骨架连线出点，当前会同时做外轮廓采样、躯干面填充、四肢带状填充和头部椭圆填充，尽量接近整块人体扫描。
- 身体轮廓现在还会基于姿态关键点构建一个 silhouette 多边形，并在多边形内部直接采样粒子，身体中心区域会比之前更实心。
- 为避免姿态轻微抖动导致扫描层频繁重建，身体填充数量改成稳定配额，并且同模式更新时优先复用现有粒子，只刷新目标位置。
- 为优先保证扫描流畅度，当前已明显下调基础粒子密度与填充数量，并为 pose / face 生成结果设置总量上限，避免轮廓捕获时粒子数瞬时冲高。
